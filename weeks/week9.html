Daily Entries

Day 1 - Monday

Today I was introduced to my main project for the remainder of my training period - the Super Admin Portal.

Activities:

  • Attended the project kickoff meeting with the team
  • Received an overview of the Super Admin Portal project
  • Discussed the project's purpose and importance for Wits Innovation Lab
  • Took notes on the main features and modules required

Project Overview:

The Super Admin Portal will be an internal tool for Wits to manage their software products, clients, and subscriptions. It will help track which clients are using which products, manage pricing plans, and handle invoicing. I'll be working as a backend developer using Node.js, TypeScript, and MongoDB.

Day 2 - Tuesday

Today we delved deeper into the project requirements and started planning the database structure.

Activities:

  • Analyzed the detailed requirements document
  • Identified the main entities needed for the database
  • Started drafting initial schema designs for products, plans, and cycles
  • Discussed potential challenges and approaches with my mentor

Key Insights:

The database design will be crucial for this project. We need to create flexible schemas that can accommodate different product types, pricing plans, and subscription cycles. The relationships between these entities will be complex and require careful planning.

Day 3 - Wednesday

Today we focused on understanding the client module requirements and planning the API structure.

Activities:

  • Analyzed client management requirements
  • Drafted the client schema with necessary fields
  • Planned the RESTful API endpoints for client management
  • Discussed how clients will be linked to products and subscriptions

Key Decisions:

We decided to use a comprehensive client schema that includes contact information, billing details, and subscription history. The API will support all CRUD operations plus additional functionality for activating/deactivating clients and managing their subscriptions.

Day 4 - Thursday

Today we explored the invoicing requirements and started planning the project structure.

Activities:

  • Analyzed the invoicing module requirements
  • Researched PDF generation and email sending in Node.js
  • Planned the project folder structure and architecture
  • Set up the initial project repository

Technical Decisions:

We decided to use a modular architecture with separate controllers for each main feature. For PDF generation, we'll explore libraries like PDFKit or html-pdf. For email functionality, we'll use Nodemailer. The project will follow a clean architecture pattern with clear separation of concerns.

Day 5 - Friday

Today we finalized the project plan and prepared for development to begin next week.

Activities:

  • Created a detailed project roadmap with milestones
  • Set up the development environment with Node.js, TypeScript, and MongoDB
  • Initialized the project with necessary dependencies
  • Created the basic Express server structure
  • Had a final planning meeting with the team

Next Steps:

Starting next week, I'll begin implementing the core functionality of the Super Admin Portal. The first task will be to set up the database connection and create the initial schemas for products, plans, and cycles. Then I'll develop the basic CRUD operations for these entities.

End of week reflection: This week has been exciting as I've been introduced to a real-world project that will utilize all the skills I've learned so far. The Super Admin Portal is a complex application with many interconnected parts, but breaking it down into modules makes it more manageable. I'm looking forward to applying my Node.js and TypeScript knowledge to build a robust backend for this system. The planning phase has been valuable for understanding the full scope of the project before diving into development.
