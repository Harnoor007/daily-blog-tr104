Week 16: Super Admin Portal - Invoice Module as a Package

Back to Weeks Week 15 Week 17

Project: Super Admin Portal

Current Focus

This week, I focused on refactoring the invoice module into a standalone package that can be reused across different projects at Wits Innovation Lab.

Key Features Implemented

  • Extracting invoice functionality into a reusable package
  • Setting up the package on GitLab for version control
  • Creating different invoice templates (Indian/International)
  • Implementing S3 integration for PDF storage
  • Building a clean API interface for the invoice package

Daily Entries

Day 1 - Monday

Planned the extraction of the invoice module and set up the package structure.

Tasks completed:

  • Analyzed the current invoice module to identify components for extraction
  • Designed the package structure and API interface
  • Set up the package repository on GitLab
  • Created the initial package configuration with TypeScript

Challenges faced:

Determining what should be included in the package versus what should remain in the main application required careful consideration. I needed to ensure a clean separation of concerns while maintaining all necessary functionality.

Day 2 - Tuesday

Extracted core invoice generation functionality into the package.

Tasks completed:

  • Extracted invoice data processing functions
  • Implemented invoice number generation utility
  • Created interfaces for invoice data structures
  • Set up unit tests for core functions

Challenges faced:

Ensuring that the extracted code was generalized enough to work in different contexts while still providing all necessary functionality was challenging. I had to carefully refactor code that had dependencies on the main application.

Day 3 - Wednesday

Implemented multiple invoice templates and PDF generation.

Tasks completed:

  • Created template for Indian invoices with GST support
  • Implemented international invoice template
  • Developed template selection logic based on client location
  • Optimized PDF generation for consistent rendering

Challenges faced:

Creating invoice templates that complied with different regional requirements (especially Indian GST regulations) required research and careful implementation. Ensuring consistent PDF rendering across different data scenarios was also challenging.

Day 4 - Thursday

Integrated AWS S3 for invoice storage and implemented email functionality.

Tasks completed:

  • Set up AWS S3 integration for PDF storage
  • Implemented secure URL generation for invoices
  • Extracted email sending functionality into the package
  • Created configurable email templates

Challenges faced:

Implementing secure S3 integration required careful handling of credentials and permissions. I needed to ensure that the package could work with different S3 bucket configurations while maintaining security.

Day 5 - Friday

Finalized the package and integrated it back into the main application.

Tasks completed:

  • Completed documentation for the invoice package
  • Published the package to GitLab registry
  • Integrated the package into the Super Admin Portal
  • Updated existing invoice-related code to use the package
  • Tested the integration thoroughly

End of week reflection: This week I successfully refactored the invoice module into a standalone package that can be reused across different projects. This was a valuable exercise in code organization and modularization. I learned a lot about package development, versioning, and integration. The invoice package now provides a clean, well-documented API that handles invoice generation, PDF creation, S3 storage, and email sending. The multiple template support adds flexibility for different regional requirements. This refactoring will make maintenance easier and allow the invoice functionality to be reused in other projects at Wits Innovation Lab.
